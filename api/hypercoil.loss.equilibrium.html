Equilibrium and SoftmaxEquilibrium#

Loss functions to favour equal weight across one dimension of a parcellation tensor.

Equilibrium

The equilibrium loss of a parcellation tensor \(A\) is defined as

\(\mathbf{1}^\intercal \left[\left(A \mathbf{1}\right) \circ \left(A \mathbf{1}\right) \right]\)

The parcel equilibrium is principally designed to operate on parcellation tensors. A parcellation tensor is one whose rows correspond to features (e.g., voxels, time points, frequency bins, or network nodes) and whose columns correspond to parcels. Element \(i, j\) in this tensor accordingly indexes the assignment of feature \(j\) to parcel \(i\). Examples of parcellation tensors might include atlases that map voxels to regions or affiliation matrices that map graph vertices to communities. It is often desirable to constrain feature-parcel assignments to \([0, k]\) for some \(k\) and ensure that the sum over each feature’s assignment is always \(k\). (Otherwise, the unnormalised loss could be improved by simply shrinking all weights.) For this reason, we can either normalise the loss or situate the parcellation tensor in the probability simplex using a multi-logit (softmax) domain mapper.

The parcel equilibrium attains a minimum when parcels are equal in their total weight. It has a trivial and uninteresting minimum where all parcel assignments are equiprobable for all features. Other minima, which might be of greater interest, occur where each feature is deterministically assigned to a single parcel. These minima can be favoured by using the equilibrium in conjunction with a penalty on the entropy.

Loss function weight multiplier.

Vectors along the specified axis should correspond to the assignment probabilities of every feature to a single parcel.

Map from a tensor of arbitrary dimension to a scalar. This has no effect unless the input is a batch or block of parcellation tensors, in which case it reduces over the batch.
